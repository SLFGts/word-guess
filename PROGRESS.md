##  关键技术点

### 前后端数据流
```
用户输入 → api.guess() → 云函数 guess() → 查 rankings 表 → 对数映射 → 返回 {score, won}
                                      ↓
                            game-store.js 记录 → 本地存储 wx.setStorageSync
                                      ↓
                            setData 更新 UI → 分数条/历史列表/提示面板
```

### 状态管理
- `game-store.js`：单局游戏状态（去重+最高分）
- `wx.setStorageSync`：跨局持久化（历史记录+统计）
- 云数据库 `games` 集合：多局游戏数据（排名表+提示缓存）

### 性能优化
- 云函数实例级缓存：向量矩阵 40MB 首次加载后复用
- 提示缓存：newGame 预生成 T1/T2，getHint 秒返
- 本地存储：历史记录最多 50 条，防止膨胀

## 📝 注意事项

1. **字体加载**：小程序不支持包内字体，必须用云存储或 CDN URL
2. **云函数冷启动**：newGame 首次调用 5-10s，后续 1-3s（实例复用）
3. **LLM 成本**：每局约 2-3 次调用（smart_pick+T1+T2），MVP 阶段约¥10/月
4. **向量数据**：`vectors-f32.bin` 约 40MB，需上传到 newGame 云函数目录

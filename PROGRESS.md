### 2. 后端绑定（双模式架构）

| 云函数 | Mode A (vector) | Mode B (embedding) | 状态 |
|--------|-----------------|---------------------|------|
| `newGame` | 下载向量→LLM 选词+预生成3级提示→存数据库 | 选词→调 Embedding API→存目标词向量 | ✅ |
| `guess` | 查排名表→对数映射→返回分数 | 调 Embedding API→1v1 余弦相似度→返回原始相似度 | ✅ |
| `getHint` | 返回3级递进提示（T1类别/T2范围/T3特征） | 同左 | ✅ |
| `getAnswer` | 返回目标词答案 | 同左 | ✅ |

**切换方式**：云开发控制台设置环境变量 `SIMILARITY_MODE`（`vector` 或 `embedding`）

### 3. 提示系统（3级递进）

从原来的2级提示升级为3级递进漏斗：

| 级别 | 内容 | 示例 |
|------|------|------|
| T1 类别 | 词性+所属大类（≤10字） | 「一种水果」「一种哺乳动物」 |
| T2 范围 | 产地/体型/颜色/功能等（≤20字） | 「常见于热带地区」「属于犬科」 |
| T3 特征 | 鲜明具体特征（≤20字） | 「果皮有刺且气味浓烈」「尾巴蓬松会竖起来」 |

设计要点：
- 三条提示形成「类别→范围→特征」的递进漏斗
- LLM 一次调用预生成3条提示，玩家点击秒返
- 泄露判定：不含目标词2字及以上子串
- 每局每级仅可点一次

## 技术架构

### 双模式对比

| 指标 | Mode A（vector） | Mode B（embedding） |
|------|-------------------|------------------------|
| 冷启动 | 10-20 秒（下载 41MB） | **0 秒** |
| 每次猜词 | < 100ms | 200-500ms |
| 分数形式 | 排名映射百分比（78.8%） | 原始余弦相似度（0.7234） |
| 词库限制 | 53219 词（OOV 提示） | 无限制（任意词） |
| 月费 | ¥1 | ≈ ¥0 |
| 适用场景 | 正式环境（分数更精准） | 调试/快速验证 |

### 数据流

```
用户输入 → api.guess() → 云函数 guess()
                            ↓
                    读 similarityMode 字段
                            ↓
              ┌─────────────┴─────────────┐
              ↓                           ↓
         Mode A: vector              Mode B: embedding
         查 rankings 表              调 Embedding API
         对数映射→score              1v1 点积→similarity
              ↓                           ↓
              ─────────────┬─────────────┘
                            ↓
                    返回前端显示
```

### 状态管理

- `game-store.js`：单局游戏状态（去重+最高分）
- `wx.setStorageSync`：跨局持久化（历史记录+统计）
- 云数据库 `games` 集合：多局游戏数据（含 `similarityMode` 字段）
